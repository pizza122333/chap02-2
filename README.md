1. Visual Studio 프로젝트 속성에서 Debug 모드와 Release 모드의 차이

Microsoft Visual Studio에서 프로그램을 빌드할 때 Debug 모드와 Release 모드를 선택할 수 있다.

Debug 모드

Debug 모드는 프로그램의 오류를 찾고 수정하기 위한 개발용 모드이다.

특징

* 디버깅 정보 포함
* 변수 값 확인 가능
* 실행 속도가 비교적 느림
* 프로그램 크기가 큼

주요 목적
→ 프로그램 오류 수정 및 테스트


Release 모드

Release 모드는 최종 사용자에게 배포하기 위한 모드이다.

특징

* 디버깅 정보 제거
* 코드 최적화 수행
* 실행 속도가 빠름
* 실행 파일 크기가 작음

주요 목적
→ 실제 프로그램 배포


2. Visual Studio 프로젝트 속성에서 x86 모드와 x64 모드의 차이

Microsoft Visual Studio에서 프로그램을 빌드할 때 x86과 x64 플랫폼을 선택할 수 있다.

x86 모드

x86 모드는 32비트 환경에서 실행되는 프로그램을 생성하는 모드이다.

특징

* 32비트 CPU 사용
* 최대 약 4GB 메모리 사용 가능
* 구형 시스템과 호환 가능


x64 모드

x64 모드는 64비트 환경에서 실행되는 프로그램을 생성하는 모드이다.

특징

* 64비트 CPU 사용
* 4GB 이상의 메모리 사용 가능
* 성능이 더 좋음


3. 포함 디렉토리를 설정하는 이유

프로그램에서 외부 라이브러리를 사용할 때 **헤더 파일(.h)**이 필요하다.

빌드 전에 포함 디렉토리(Include Directory)를 설정하는 이유는
컴파일러가 해당 헤더 파일의 위치를 찾을 수 있도록 하기 위해서이다.

예

```id="9n9chj"
#include <opencv2/opencv.hpp>
```

이 헤더 파일을 찾기 위해 컴파일러는 설정된 포함 디렉토리를 검색한다.

예시 라이브러리

* OpenCV


4. 라이브러리 디렉토리를 설정하는 이유

라이브러리 디렉토리는 라이브러리 파일(.lib)이 저장된 위치를 지정하는 설정이다.

빌드 과정에서 링커가 필요한 라이브러리를 찾기 위해 사용된다.

즉

* 컴파일 단계 → 헤더 파일 필요
* 링크 단계 → 라이브러리 파일 필요

예

```
opencv_world480.lib
```

이 파일을 찾기 위해 라이브러리 디렉토리를 설정한다.


5. 링커 추가 종속성을 설정하는 이유

링커 추가 종속성(Additional Dependencies)은 프로그램이 사용할 라이브러리 파일을 지정하는 설정이다.

링커는 이 설정을 통해 어떤 라이브러리를 연결할 것인지 결정한다.

예

```
opencv_world480.lib
```

이 파일이 추가되면 프로그램에서 OpenCV 기능을 사용할 수 있다.

즉

* 라이브러리 디렉토리 → 라이브러리 위치
* 추가 종속성 → 사용할 라이브러리 이름


6. DLL을 사용하기 위해 PATH에 추가하는 경로와 이유

DLL을 사용할 때 DLL 파일이 있는 폴더 경로를 환경 변수 PATH에 추가해야 한다.

대표적인 DLL 형식

* Dynamic Link Library

예

```
C:\opencv\build\x64\vc16\bin
```

이 경로에는 OpenCV DLL 파일들이 존재한다.

PATH에 추가하는 이유

Microsoft Windows는 프로그램 실행 시 DLL을 찾기 위해 다음 위치를 검색한다.

1. 실행 파일 폴더
2. 시스템 폴더
3. 환경 변수 PATH에 등록된 폴더

따라서 DLL 경로가 PATH에 등록되어 있지 않으면 운영체제가 DLL을 찾지 못해 프로그램이 실행되지 않는다.

예 오류

DLL file not found

그래서 DLL이 있는 폴더 경로를 PATH 환경 변수에 추가해야 한다.
